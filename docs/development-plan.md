# 店小秘主控的 Temu AI 自动上品系统开发规划

## 1. 项目目标

构建一套以 AI 为核心、以店小秘为主控台的 Temu 自动上品系统，覆盖选品采集、核价、内容生成、店小秘刊登、任务巡检与日志追踪。第一阶段先打通最小业务闭环：候选商品进入系统后，AI 生成上品决策与执行计划，浏览器插件优先服务店小秘页面流程，Temu 页面仅作为辅助采集和核验入口。

## 2. 产品定位

- 控制台：商品池、任务队列、规则配置、日志查看
- AI 服务：选品评分、核价、文案生成、执行编排
- 浏览器插件：店小秘页面识别、Temu 页面采集、自动填表、执行反馈
- 共享模型：商品、任务、核价结果、执行步骤、风控状态

## 2.1 主控策略

- 店小秘：主控系统，负责采集箱、产品编辑、刊登、店铺授权、流程串联
- Temu：辅助入口，负责官方站点采集、页面核验和必要的异常补录
- AI 服务：在店小秘业务流上生成选品、核价、文案和执行策略

## 2.2 运营约束

- 默认入口只有 `无人值守主流程`：使用店小秘已采集商品，自动修改、处理图片、保存并点发布。
- manual / review / repair / selector / import 等工具只能在 `高级区` 默认隐藏，不作为日常员工主路径。
- 永久保留的人工步骤只有 Temu 核价确认和最终上架确认。
- 新增任何人工步骤必须写明替代自动化计划和下线时间；没有预算说明的人工步骤不能进入默认流程。
- `repair-*` 只属于故障恢复，不进入日常 KPI。
- AI 功能必须证明减少点击或减少人工判断后，才能进入默认流程。

## 3. 阶段规划

### 阶段一：最小闭环 MVP

目标：完成“导入商品 -> AI 生成上品计划 -> 插件在店小秘页面展示执行计划 -> 页面自动填充骨架”。

交付：

- Monorepo 工程结构
- 本地控制台首页与任务面板
- AI 编排服务 API
- 浏览器插件侧边栏与内容脚本
- 共享数据模型与模拟数据

### 阶段二：业务能力增强

目标：引入真实核价逻辑、规则引擎和任务管理。

交付：

- 成本、运费、利润率核价模块
- 类目属性模板
- 批量任务队列
- 失败恢复与日志审计

### 阶段三：智能化决策

目标：接入外部数据源和大模型，形成店小秘主导的自动选品与内容生成闭环。

交付：

- 候选商品采集器
- 竞品分析
- AI 文案与标题改写
- 风险解释与人工复核节点

### 阶段四：全流程代理

目标：实现自动上品、巡检、修正和复盘。

交付：

- 自动提交与状态巡检
- 驳回原因解析
- 二次修正策略
- 日报与选品复投建议

```text
  dashboard/    本地控制台

  extension/    浏览器插件

  shared/       共享类型、规则、mock
docs/
  development-plan.md
```

## 6. MVP 业务流程

1. 控制台展示候选商品
2. 调用 AI 服务生成上品计划
3. 服务返回核价、标题、卖点、执行步骤
4. 插件读取计划并在店小秘页面展示
5. 内容脚本按计划执行采集、编辑或刊登辅助动作
6. 必要时在 Temu 官方采集页面执行辅助采样或核验
7. 执行结果回传控制台日志

## 7. 核心数据对象

- ProductCandidate
- PricingAnalysis
- ListingDraft
- PublishTask
- ExecutionStep
- RiskAlert

## 8. 风险控制原则

- 默认保存草稿，不直接最终提交
- 所有高风险动作保留人工确认开关
- 执行步骤可中断、可恢复、可记录
- 选择器适配器与页面逻辑分离，降低 DOM 变化影响
- 店小秘与 Temu 页面适配器分离，避免流程耦合

## 9. 当前迭代目标

阶段一~三的主体能力已落地（工程结构、控制台、核价规则、任务队列、内容生成含真实大模型交互路径）。当前迭代属于阶段四的前哨：打通真实店小秘写链路的最后阻塞墙（主题颜色/变种行、图片恢复真跑验证），完成 limit=3 真实试跑并解锁无人值守。实时进度见 [current-status.md](current-status.md) 与 [blocking-walls-diagnosis.md](blocking-walls-diagnosis.md)。

## 10. 官方集成边界

根据店小秘官方帮助文档，当前集成策略遵循以下边界：

- 店小秘支持 Temu 店铺授权后在店小秘内发布产品
- 店小秘支持 1688 等来源的关键词采集与插件采集
- Temu 财务与申报价相关采集更适合作为辅助功能，不应替代店小秘主流程
- 插件优先覆盖店小秘工作台，其次覆盖 Temu 官方采集入口
